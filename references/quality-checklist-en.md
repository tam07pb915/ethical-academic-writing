# Quality Checklists (English)

## 1. Ethics Checklist (Verify Before Submission)

- [ ] AI did not generate the full text: I conceived and outlined the content myself
- [ ] All ideas are my own: research conception, hypotheses, and interpretations belong to me
- [ ] I reviewed all AI suggestions: no uncritical copy-paste of AI output
- [ ] Expressions match my proficiency level: within Augmented Competence range
- [ ] My voice is preserved: the paper does not read as "anyone could have written this"

## 2. Pre-Submission Checklist

### Content and Structure
- [ ] RQs are answered in Results and Discussion
- [ ] Logical flow from Introduction → Methods → Results → Discussion is consistent
- [ ] No duplication or contradiction between sections
- [ ] Abstract matches main text in numbers and conclusions
- [ ] Abstract contains no information absent from the main text

### Style and Formatting
- [ ] British/American spelling is consistent throughout
- [ ] Number formatting is consistent (spell out below 10, spell out at sentence start)
- [ ] All abbreviations are spelled out at first use
- [ ] First-person usage is consistent (no we/I mixing)
- [ ] Hedging strength is consistent for claims of equal confidence
- [ ] No unnatural repetition of AI-typical formulaic expressions

### Figures, Tables, and References
- [ ] Every figure/table is referenced in the text
- [ ] Figure/table captions are self-contained
- [ ] Every in-text citation appears in the reference list
- [ ] Every reference list entry is cited in the text
- [ ] DOIs are verified via Crossref
- [ ] Citation style follows the target journal's requirements

### Submission Requirements
- [ ] Word count is within the journal's limit
- [ ] Formatting (font, spacing, margins) follows the journal's guidelines
- [ ] AI-use disclosure is included (if required)
- [ ] Cover letter is prepared
- [ ] Conflict of interest disclosure is included

## 3. Eight Failure Patterns

| # | Pattern | Warning sign | Remedy |
|---|---------|-------------|--------|
| 1 | Full delegation | Requesting paper generation without a draft | Write L1 outline first |
| 2 | Fabricated citations | Using AI-generated references without verification | Verify every DOI via Crossref |
| 3 | Tense mixing | Inconsistent tense between AI and self-written parts | Run tense check per section |
| 4 | Over-hedging | Excessive may/might/could/possibly | Evaluate hedging necessity per claim |
| 5 | Escalating dependency | Started with proofreading, now generating paragraphs | Set personal rules at the start |
| 6 | Style chimera | AI-generated parts noticeably more fluent | Run full-text style consistency check |
| 7 | Uncritical adoption | Accepting 100% of AI suggestions | Always choose from options |
| 8 | Security risk | Inputting unpublished data into AI | Avoid entering sensitive information |

## 4. Fact-Checking Procedure

### Reference Verification Flow

1. Search author + year + title in Google Scholar
2. Verify DOI via Crossref (https://search.crossref.org/)
3. Visit the DOI URL to confirm the paper exists
4. Confirm that the cited content matches the source

### Four Types of AI Hallucination

| Type | Description | Risk |
|------|-------------|------|
| Complete fabrication | Author, title, and DOI are all fictional | Critical |
| Partial fabrication | Real author name + fictional paper title | High (hard to detect) |
| Distortion | Real paper summarized inaccurately | Medium |
| Context misfit | Accurate information cited in wrong context | Medium |

## 5. Phased AI Adoption

| Phase | Scope | Risk Level |
|-------|-------|------------|
| Phase 1 | Checking and proofreading (grammar, spelling) | Low |
| Phase 2 | Expression improvement (paraphrasing, style adjustment) | Low–Medium |
| Phase 3 | Structure and logic support (outlining, structure checks) | Medium |
| Phase 4 | Draft support (L1 → English conversion) | Medium–High (requires disclosure) |

Start from Phase 1 and expand gradually as you gain experience.
